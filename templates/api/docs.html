Documentación de la API

ShirkaAI es compatible con el formato de la API de OpenAI

Mis tokens

Base URL

{{ base_url }}/v1

Autenticación

Incluye tu token en el header Authorization de cada solicitud:

Authorization: Bearer sk-shirka-<tu-token>

Genera tus tokens en API → Tokens.

GET /v1/models

Lista los modelos descargados y listos en ShirkaAI.

Ejemplo

curl {{ base_url }}/v1/models \
  -H "Authorization: Bearer sk-shirka-<tu-token>"

Respuesta

{
  "object": "list",
  "data": [
    {
      "id": "qwen2.5-0.5b",
      "object": "model",
      "created": 1716000000,
      "owned_by": "shirkaai",
      "name": "Qwen 2.5 - 0.5B",
      "status": "active"
    }
  ]
}

POST /v1/chat/completions

Genera una respuesta de chat. Mismo formato que OpenAI.

Body (JSON)

Campo Tipo Descripción
modelstringID del modelo (e.g. qwen2.5-0.5b). Si no existe, se usa el modelo activo.
messagesarrayRequerido. Lista de mensajes {role, content}. Roles: system, user, assistant.
max_tokensintegerMáximo de tokens a generar. Default: 512.
temperaturefloatCreatividad (0.0 – 2.0). Default: 0.7.

Ejemplo curl

curl {{ base_url }}/v1/chat/completions \
  -H "Authorization: Bearer sk-shirka-<tu-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5-0.5b",
    "messages": [
      {"role": "system", "content": "Eres un asistente útil."},
      {"role": "user",   "content": "¿Cuál es la capital de México?"}
    ],
    "temperature": 0.7,
    "max_tokens": 256
  }'

Respuesta

{
  "id": "chatcmpl-a1b2c3d4e5f6",
  "object": "chat.completion",
  "created": 1716000000,
  "model": "qwen2.5-0.5b",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "La capital de México es Ciudad de México."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 18,
    "completion_tokens": 9,
    "total_tokens": 27
  }
}

Uso con el SDK oficial de OpenAI

Cambia sólo el base_url — el resto del código es idéntico:

from openai import OpenAI

client = OpenAI(
    api_key="sk-shirka-<tu-token>",
    base_url="{{ base_url }}/v1",
)

response = client.chat.completions.create(
    model="qwen2.5-0.5b",
    messages=[
        {"role": "user", "content": "Hola, ¿cómo estás?"},
    ],
)
print(response.choices[0].message.content)

Códigos de error

Código Significado
401Token inválido o revocado.
400Cuerpo JSON inválido o falta el campo messages.
404Modelo no encontrado o no está listo.
500Error interno al generar la respuesta.
